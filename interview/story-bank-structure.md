# 面试故事库结构建议

## 目标

把你过往经历从“零散经历”整理成“可复用故事模块”，便于：
- 自我介绍
- 回答行为面试题
- 回答专业面试题
- 针对 analyst / consulting / hybrid 岗位切换叙事

## 推荐故事主库

### story_01_fujikura_market_intel
- 核心主题：光通信市场研究与总部战略支持
- 适合岗位：analyst / consulting / strategy / market intelligence
- 亮点：
  - 输出中国光通信市场结构化研判
  - 覆盖 FTTH / 5G / DCI / 运营商采购
  - 支撑日本 / 欧洲总部决策

### story_02_fujikura_data_center_entry
- 核心主题：发现并验证数据中心新机会
- 适合岗位：business analysis / commercial strategy / BDM
- 亮点：
  - 识别银行与政务数据中心机会
  - 推动新业务落地
  - 有实际收入结果

### story_03_fujikura_pipeline_model
- 核心主题：用结构化分析支持商业判断
- 适合岗位：analyst / commercial analyst / strategy
- 亮点：
  - 搭建 2 亿+ RMB 项目管道模型
  - 识别优先机会与趋势
  - 把分析转成管理决策

### story_04_omron_zero_to_one
- 核心主题：从 0 到 1 开新市场
- 适合岗位：BDM / hybrid / growth / consulting
- 亮点：
  - 独立建立华中市场
  - 首年出成果
  - 体现执行与判断力

### story_05_japan_hq_bridge
- 核心主题：跨文化汇报与总部协同
- 适合岗位：外资 / 欧美外企 / 日企 / 咨询
- 亮点：
  - 中英日三语
  - 可对接总部
  - 能把中国市场复杂信息翻译成管理层可用语言

### story_06_key_account_insight
- 核心主题：从客户关系走向客户洞察
- 适合岗位：hybrid / consulting / market-facing analyst
- 亮点：
  - 不只是维护客户
  - 能从客户需求中抽象出行业趋势和机会

## 每个故事建议模板

### 基本信息
- story_id
- 标题
- 适用岗位
- 适用语言

### STAR 结构
- situation
- task
- action
- result

### 面试可复用版本
- 30秒版
- 90秒版
- 深挖版

### 标签
- industry
- capability
- language
- evidence_type

## 建议优先写好的 3 个故事

1. `fujikura_market_intel`
2. `fujikura_pipeline_model`
3. `omron_zero_to_one`

## 为什么这 3 个最优先

- 能覆盖 analyst
- 能覆盖 consulting
- 能覆盖 hybrid
- 同时保留你真实最强的行业背景

## 后续使用方法

每次面试前，只做两件事：
- 先选 3 个主故事
- 再按岗位改成对应版本

这样你就不会每次从头组织表达
